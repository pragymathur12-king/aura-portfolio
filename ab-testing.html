Video Engagement Analytics

A/B testing + funnel diagnosis

A story of moving from guesswork to measurable product behavior—using experiments to improve engagement where it matters.

Role
Product Manager (Case Study)
Product Type
Experimentation + Funnel Analytics
Category
Experimentation & Growth
Duration
Proposed KPI: 4–7 Weeks
Tools Used
Experiment planning, Funnel instrumentation
Skills Demonstrated
Causal hypotheses, Guardrails, Segmentation thinking
Reading Time
Proposed KPI: 5–7 Minutes

🎯 Problem

User Pain

  • Users clicked, but didn’t stick to the content.
  • Key engagement moments dropped mid-funnel.
  • Teams lacked a clear causal story for improvements.

💡 Insights

  • The biggest churn happened before the “first value” moment.
  • Small UI framing changes influenced watch-through behavior.
  • Engagement metrics needed segmentation (new vs returning users).

🚀 Solution

  • Experiment plan built around funnel step hypotheses.
  • Variant design focused on surfacing “first value” sooner.
  • Experiment success tied to watch-through and retention proxies.

🧩 Execution

How it works

  1. Instrument the funnel: click → start → first value → watch-through.
  2. Generate hypotheses from drop-offs and session context.
  3. Run A/B tests with guardrails (no negative experience metrics).
  4. Iterate by learning: winners become defaults, losers become insights.

Experiment focus

  • Trigger timing for recommendations.
  • Thumbnail + title framing to set expectations.
  • Onboarding nudges for new users.

📈 Impact

  • Improved watch-through by addressing the earliest engagement leak.
  • More reliable decision-making with experiment-backed outcomes.
  • Clearer product narrative for future iterations.

📚 Learnings

  • Good metrics connect product behavior to business outcomes.
  • Segmentation prevents misleading averages.
  • Every test should leave behind a reusable learning artifact.
